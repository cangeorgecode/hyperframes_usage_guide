# hyperframes_usage_guide
My own Hyperframes usage guide with Hermes Agent

## How does Hyperframes work?
Basically takes prompts and translate it into HTML, GSAP. Using predefined keywords / vocabs, structured prompt, one could automate faceless content creation quite easily. 

## Sources 
Source: URL (website)  
How it feeds into the video: I capture screenshots, extract colors/fonts, use sections as scene content  
Example: https://sidequest.meme  


Source: Screenshots (.png, .jpg)  
How it feeds into the video: Dropped into assets/, used as B-roll backgrounds or scene focal points  
Example: Dashboard screenshots, product pages  

Source: Screen recordings (.mp4)  
How it feeds into the video: Used as <video class="scene clip"> — full-motion B-roll, product demos  
Example: Terminal recording, app walkthrough  

Source: CSV / data  
How it feeds into the video: Fed into data-chart or apple-money-count blocks as animated visualizations  
Example: Revenue data, user growth numbers  

Source: Transcripts / articles  
How it feeds into the video: Turned into VO script, then TTS narration  
Example: Blog post → video script  

Source: GitHub repos  
How it feeds into the video: README content + screenshots + terminal blocks  
Example: Product demo from a repo  
 
Source: Reddit threads / tweets  
How it feeds into the video: Social proof blocks (reddit-post, x-post)  
Example: "Look what people are saying"  

Source: Pexels stock footage  
How it feeds into the video: Downloaded via API, used as B-roll
